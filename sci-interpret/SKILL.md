# SCI 文献解读技能

将原版SCI文献按标准化工作流解读，输出公众号可用的文献解读文档（.docx格式，WPS可直接打开使用）。

## 使用方式

```
/sci-interpret E:\文献解读\02-\某文献.pdf
```

## 输出

文件保存至：`E:\文献解读\` 对应子目录下
命名格式：`{文献简称}-{YY-MM-DD}.docx`
示例：`暴露组学-26-04-06.docx`

---

## 全局格式规范（贯穿所有工作流）

### 字体与样式
- 全文所有文字统一使用「思源黑体 Normal」字体
- 包括：标题、正文、图注、技术路线图中的文字，无一例外
- 技术路线图生成时，matplotlib/PIL 必须加载思源黑体字体文件渲染中文

### 文档格式（WPS兼容）
- 输出为标准 .docx 格式，确保WPS可直接打开、复制使用，无须二次整理
- 页面设置：A4纸，页边距上下2.54cm、左右3.18cm（Word/WPS默认值）
- 段落间距：段前0、段后0、行距1.5倍
- 标题使用 Heading 3 样式（同样设置为思源黑体）
- 正文使用 Normal 样式
- 图片插入宽度：6英寸（适配A4页面）
- 图注格式：「图X. 图注内容」
- 基本信息部分：标签名与内容在同一行，用冒号分隔（如「发表期刊：Environmental Science & Technology」）

### python-docx 字体设置方式
```python
from docx import Document
from docx.shared import Pt
from docx.oxml.ns import qn

doc = Document()

# 设置默认字体为思源黑体
style = doc.styles['Normal']
style.font.name = 'Source Han Sans CN'  # 思源黑体
style.font.size = Pt(11)
style.element.rPr.rFonts.set(qn('w:eastAsia'), 'Source Han Sans CN')

# Heading 3 同样设置
h3_style = doc.styles['Heading 3']
h3_style.font.name = 'Source Han Sans CN'
h3_style.font.size = Pt(14)
h3_style.element.rPr.rFonts.set(qn('w:eastAsia'), 'Source Han Sans CN')
```

### matplotlib 中文字体设置方式
```python
import matplotlib.pyplot as plt
from matplotlib import font_manager

# 加载思源黑体（根据系统实际路径调整）
font_path = 'C:/Windows/Fonts/SourceHanSansCN-Normal.otf'  # 或 .ttf
font_prop = font_manager.FontProperties(fname=font_path)
plt.rcParams['font.family'] = font_prop.get_name()
plt.rcParams['axes.unicode_minus'] = False
```

如果系统中找不到思源黑体字体文件，先用以下命令搜索：
```bash
find /c/Windows/Fonts -iname '*sourcehan*' -o -iname '*思源*' -o -iname '*noto*sans*' 2>/dev/null
```
如果确实没有，退而使用「微软雅黑」作为备选。

## 工作流执行步骤

### 工作流【1】：提取文献基本信息

阅读PDF全文，提取以下10项信息：

1. 公众号推文标题（格式参考：「期刊名 (IF XX) | 核心发现关键词+机制/方法亮点」）
2. 英文标题
3. 中文标题（加书名号《》）
4. 发表期刊
5. 影响因子
6. 发表时间
7. 研究单位（通讯作者所在单位+团队负责人）
8. 文章简介（300-500字，包含以下要素）：
   - 第一段：研究背景与问题（该领域现状、未解决的关键问题）
   - 第二段：必须包含「时间+团队+期刊+论文标题」句式，如：「2026年3月复旦大学环境科学与工程系方明亮教授团队在Environmental Science & Technology（IF=10.1）上发表了题为"Mechanistic Insights into Perfluoroalkyl Substance-Induced Thyroid Hormone Disruption during Pregnancy：Evidence from Metabolomic Mediation Analysis"的研究论文。」
   - 第三段：该研究的核心发现与意义
9. 研究结果标题列表（Result部分所有小标题，用1、2、3编号）
10. 研究结论（全文最后两段，翻译成中文）

确认10项信息无误后，进入工作流【2】。

---

### 工作流【2】：翻译研究结果1-3

将原文Result中第1、2、3个小标题及其完整内容逐条翻译为中文。

要求：
- **直译原文，禁止对内容进行任何润色、改写或删减**，忠实还原原文表述
- 每条翻译必须带上对应的小标题（中文翻译），格式为「编号. 中文标题」
- 保留所有数据、统计值、图表引用（图/表标注与原文完全一致）
- 专业术语保留英文缩写+中文全称
- 翻译准确，不遗漏关键信息
- 每个结果之间空一行分隔

---

### 工作流【3】：翻译研究结果4-6

将原文Result中第4、5、6个小标题及其完整内容逐条翻译。

要求：
- **直译原文，禁止对内容进行任何润色、改写或删减**，忠实还原原文表述
- 每条翻译必须带上对应的小标题（中文翻译），格式为「编号. 中文标题」
- 保留所有数据、统计值、图表引用（图/表标注与原文完全一致）
- 专业术语保留英文缩写+中文全称

规则：
- 如果没有结果6，只翻译到结果4、5即完成本步
- 如果存在结果7、8、9中的任意一项，进入工作流【4】
- 如果不存在结果7、8、9，直接进入工作流【5】

---

### 工作流【4】：翻译研究结果7-9

将原文Result中第7、8、9个小标题及其完整内容逐条翻译（如有）。

要求：
- **直译原文，禁止对内容进行任何润色、改写或删减**，忠实还原原文表述
- 每条翻译必须带上对应的小标题（中文翻译），格式为「编号. 中文标题」
- 保留所有数据、统计值、图表引用（图/表标注与原文完全一致）
- 专业术语保留英文缩写+中文全称

完成后进入工作流【5】。

### 工作流【5】：精简研究结论（上半部分）

将所有研究结果的核心结论进行精简重写。

写作风格参考（必须严格遵循此风格）：

```
示例1（生化机制类——同一动机、多方法验证）：
已知赖氨酸L-乳酸化（Kₗ₋ₗₐ）是组蛋白上的主要修饰，而非赖氨酸D-乳酸化（Kd₋ₗₐ）或N-ε-（羧乙基）-赖氨酸（Kce）。
研究人员通过"重标"培养基（含K8 [¹³C₆，¹⁵N₂]标记L-赖氨酸）培养人MCF-7细胞，富集组蛋白修饰肽段后，与"轻标"（K0 [¹²C₆，¹⁴N₂]）Kₗ₋ₗₐ、Kd₋ₗₐ和Kce标准肽混合，进行HPLC-MS/MS分析（图4a）。结果显示，细胞来源肽段与Kₗ₋ₗₐ/D₋ₗₐ标准肽保留时间及碎片模式一致，与Kce标准肽不一致，表明组蛋白主要修饰为Kₗ₋ₗₐ/D₋ₗₐ而非Kce（图4b）。
为明确是Kₗ₋ₗₐ还是Kd₋ₗₐ，研究人员将富集肽段消化为氨基酸，加入"轻标"标准品后经MTPA-Cl手性衍生化，HPLC-MS/MS分析显示，细胞来源产物与Kₗ₋ₗₐ产物共洗脱，证明组蛋白乳酸化主要为L型（图4c-d）。
综上数据，细胞组蛋白的三种赖氨酸乳酸化结构异构体中，Kₗ₋ₗₐ是主要异构体。

示例2（动物代谢+人群验证类——不同动机、各自展开）：
急性热暴露可短暂增强能量消耗，但慢性热应激的长期代谢后果尚不清楚。为探究这一问题，研究人员将8周龄雄性小鼠随机分为热应激（HS, 37°C, 6h/天×1周）、配对喂养（PF）和室温（RT）三组，经2周静息期恢复后给予HFD喂养10周（图1A）。结果显示，HS小鼠在HFD喂养后出现显著体重增加、糖耐量受损和胰岛素抵抗（图1B-1D），伴随eWAT和iWAT脂肪量增加及脂肪细胞肥大（图1E和1F）。
进一步检测发现，HS小鼠eWAT中脂解和脂肪酸氧化通路显著受抑，ATGL、p-HSL、TH和PPARα蛋白水平降低，而脂肪合成蛋白FASN和SCD1无显著变化（图1G和S1I）。HFD喂养4周时（体重尚未分化），HS小鼠已表现出能量消耗降低和RER升高（图1H和1I），eWAT基础甘油释放减少但β3激动剂诱导的脂解不受影响（图1J和1K），同时脂肪组织NE水平显著低于PF小鼠（图1L），提示交感神经驱动受损而非脂肪细胞内在缺陷是脂解能力下降的主因。
在人群层面，研究人员招募了长沙外卖骑手队列进行前瞻性随访（图1M），发现热暴露后骑手的代谢指标同样出现恶化，验证了热应激对人类代谢健康的不利影响。
```

精简原则：
- 每个结果按以下三层结构组织，每层之间逻辑连贯、自然衔接：
  ① 背景/动机前缀（1-2句）：交代为什么做这个实验，或该实验要回答什么问题（如"为明确是X还是Y"、"鉴于A在B中的作用尚不清楚"、"已知X是Y的主要修饰"）
  ② 实验方法（1-2句）：具体描述研究人员做了什么操作，包含关键技术细节（如细胞系、处理条件、检测手段），方法后紧跟图标引用
  ③ 结果结论（1-2句）：结果显示了什么，得出什么结论，紧跟图标引用
- 如果一个结果中包含多个子实验，相同目的的子实验用同一动机、不同实验方法→对应的结论→对应的图标；如果每个子实验的动机都不同，则每个子实验都按「动机→方法→结论+图标」的节奏展开，最后用1~2句话总结该结果的核心结论
- 删除纯描述性冗余（如重复的分组说明），但保留关键数据、统计值和所有图/表引用
- 整体文字量：每个结果精简后约300~600字左右（视原文复杂度调整），不能只有一句话结论+图标的干巴结构；如果原文非常复杂，以保证每个结果的三层结构组织为第一原则，最高可以超过700字以上
- **图/表引用标注必须紧跟对应内容，与原文完全一致，禁止自行修改、合并或删减任何图标引用**
  - 正确示例：「...进行HPLC-MS/MS分析（图4a）。结果显示...（图4b）。」
  - 错误示例：将原文的（图4a）（图4b）合并为（图4a-b），或删除某个图标引用

如果一次无法完成所有结果的精简，完成前半部分后进入工作流【6】。
如果一次完成了全部精简，直接进入工作流【7】。

---

### 工作流【6】：精简研究结论（下半部分）

继续工作流【5】未完成的部分，用相同风格精简剩余研究结果。

要求：
- 精简风格、图标引用规则与工作流【5】完全一致
- **图/表引用标注必须紧跟对应内容，与原文完全一致，禁止自行修改、合并或删减**

完成后进入工作流【7】。

### 工作流【7】：提取图片并翻译图注

从原PDF中提取所有图片（Figure），并翻译每张图对应的图注（Figure Legend）。

执行方式：
1. 使用PyMuPDF(fitz)从PDF中提取高清图片，保存到与输出docx同目录
2. 翻译每张图的图注为中文
3. 将图片和图注按结果归属插入文档对应位置

归属规则：
- 结果1中引用了图1、图2 → 该结果下方放图1、图2及其图注
- 结果1中只引用了图1 → 该结果下方只放图1和图1图注
- 以此类推

md格式要点：
- 科学问题要点出既往方法的具体局限（如16S无法解析功能、非靶向无法定量等）

#### 第二步：生成pptx流程图

基于md内容，使用python-pptx生成可编辑的技术路线.pptx文件。

#### pptx布局规范（16:9宽屏单页水平流程图）

```

│  顶部横幅：论文标题 + 作者信息（深色背景白字）                    │

│ 科学问题  │ 研究样本  │ 核心方法                               │
│ (白底卡片) │ (白底卡片) │ (白底卡片)                             │
├──────────┴──────────┴──────────────────────────────────────┤
│  Step1 → Step2 → Step3 → Step4 → StepN（水平流程）           │

```

- 幻灯片尺寸：16:9（Inches(16) x Inches(9)）
- 背景色：#F5F7FA
- 配色方案：
  - 标题栏/结论栏：深蓝 #1B3A5C 背景 + 白色文字
  - 三卡片：白底 + 深蓝边框
  - 每个Step使用不同主题色（teal/coral/purple/green/amber等），包含：
    - 色块标签（Step编号）
    - 白底标题框（带主题色边框）
    - 浅色方法框
    - 浅黄结果框（#FFF8E1）
  - 步骤间：灰色右箭头连接
  - 结论栏：金色 #F1C40F 高亮核心通路
- 字体：Microsoft YaHei（备选 Source Han Sans CN）
- 所有文本框、标签、箭头均为独立可编辑元素
- 文本框设置 word_wrap=True
- Step数量根据论文实际内容调整（通常3-6步）

### 工作流【9】：整合输出最终文档

将所有内容按以下顺序整合为.docx文件：

① 工作流【1】的基本信息（推文标题、英文标题、中文标题、期刊、IF、时间、单位、简介）
② 研究结果（工作流【5】/【6】精简版 + 对应图片 + 图注）
③ 研究结论（工作流【1】提取的结论）

注：技术路线以独立md+pptx文件输出（工作流【8】），不插入docx。

#### 分脚本执行规范（强制执行）

由于python-docx生成含多张图片的长文档时，单脚本容易因上下文窗口限制或字符数超限而失败，**必须分两个独立Python脚本执行**：

**脚本1（gen_docx_part1.py）**：
- 创建新Document，设置字体/样式
- 写入：基本信息（10项） + 研究结论 + 研究结果前半部分（Result 1 ~ Result 3，含对应图片+图注）
- 保存为临时文件 `{文献简称}-{YY-MM-DD}_part1.docx`

**脚本2（gen_docx_part2.py）**：
- 打开Part1的临时docx（`Document(part1_path)`）
- 续写：研究结果后半部分（Result 4 ~ Result N，含对应图片+图注）
- 保存为最终文件 `{文献简称}-{YY-MM-DD}.docx`
- 删除Part1临时文件

**执行方式**：
```bash
python3 gen_docx_part1.py && python3 gen_docx_part2.py
```

**分割点选择**：
- 如果结果有5-6个：Part1写Result 1-3，Part2写Result 4-N
- 如果结果有7-9个：Part1写Result 1-4，Part2写Result 5-N
- 原则：每个脚本的文本内容不超过10000字符，确保不触发写入限制

**同样适用于pptx生成**：如果pptx脚本过长，也应写成独立.py文件后用`python3`执行，避免bash内联heredoc的引号转义问题。

输出路径：`E:\文献解读\` 对应子目录
文件名：`{文献简称}-{YY-MM-DD}.docx`

校验清单（输出前逐项确认）：
- [ ] 所有文字均为思源黑体
- [ ] WPS打开无乱码、无格式错位
- [ ] 图片清晰、位置正确
- [ ] 图注与对应结果匹配

## 推文标题公式

格式：`{期刊名}（IF{影响因子}）| {核心发现关键词}：{机制/方法亮点}，{意义/突破点}！`

## 注意事项

1. PDF读取：使用PyMuPDF(fitz)读取PDF全文及提取图片
2. 如果PDF无法直接读取（扫描版），提示用户提供可复制文本的PDF
3. 翻译质量：专业术语保留英文缩写+中文全称，如「全代谢组关联分析（MWAS）」
4. 每个工作流完成后向用户确认，再进入下一步
5. 字体回退：如果系统无思源黑体，使用微软雅黑作为备选，并告知用户
