# Daily Review（每日工作复盘）

## 触发方式

用户输入 `/daily-review` 或说「复盘」「今天总结」「daily review」时触发。

---

## 复盘输出结构（九大模块 + 自我迭代）

每次触发时，基于当天对话记录 + 用户补充信息，生成以下内容：

### 模块一：今日工作总结
- 按「主业 / 申博 / 副业」三条线分别列出今天做了什么
- 每条用一句话概括产出物和进展百分比
- 标注哪些是AI辅助完成的、哪些是纯人工的

### 模块二：战略优先级评估（新增，每次必做）
**目的**：基于3-5年目标（24W→50W），评估今天的时间分配是否合理

**评估维度**：
1. **今日时间分配**：主业X小时 / 申博X小时 / 副业X小时 / 琐碎X小时
2. **理想时间分配**（基于当前阶段）：
   - 主业：50%（每天4小时专注时间）
   - 申博：30%（每天2-3小时，5-7月重点）
   - 副业：15%（每天1-2小时，降频到每周2篇推文）
   - 琐碎：5%（批处理，每周固定1天）
3. **偏差分析**：
   - 如果副业时间>20% → 警告：副业占用过多，影响主业和申博
   - 如果琐碎时间>10% → 警告：琐碎事务碎片化，需要批处理
   - 如果主业时间<40% → 警告：主业是基本盘，优先级不够
4. **批判性建议**：
   - 哪些任务是"伪工作"（看起来在做事，但对3-5年目标无贡献）
   - 哪些任务可以外包/自动化/停止
   - 明天应该如何调整时间分配

### 模块二.5：批判性对立思维任务评估（新增，每次必做）
**目的**：用对立思维挑战今天做的每个任务，评估其对核心目标（50W年薪）的真实贡献

**评估框架**：对今天做的每个任务，提出以下5类批判性问题

#### 1. 目标贡献度质疑
**核心问题**：这个任务对50W年薪目标的贡献是什么？

**评估方法**：
- 如果停掉这个任务，3年后你的年薪会少多少？
- 这个任务是"直接贡献"（如主业项目）还是"间接贡献"（如申博）还是"几乎无贡献"（如伪工作）？
- 如果贡献<5%，为什么还要做？

**输出格式**：
| 任务 | 对50W年薪的贡献 | 贡献类型 | 是否值得继续 |
|------|----------------|---------|-------------|
| 心肌损伤SCI | 直接贡献30%（高质量交付→客户转介绍→涨价） | 直接 | ✅ 值得 |
| 公众号推文（周中） | 贡献<5%（256粉，无变现） | 几乎无 | ❌ 不值得 |

#### 2. 机会成本质疑
**核心问题**：做这个任务的时间，如果用来做别的，收益会不会更高？

**评估方法**：
- 今天做X任务花了Y小时，如果这Y小时用来做主业/申博，收益是多少？
- 对比：X任务的收益 vs 主业/申博的收益
- 如果主业/申博收益>X任务收益的3倍，说明X任务是"机会成本陷阱"

**输出格式**：
| 任务 | 投入时间 | 任务收益 | 如果做主业的收益 | 机会成本 | 结论 |
|------|---------|---------|----------------|---------|------|
| 公众号推文优化 | 3小时 | ¥0（无变现） | 心肌损伤SCI进度+10%（价值¥2000） | ¥2000 | ❌ 机会成本过高 |
| 文献解读接单 | 2小时 | ¥150 | 心肌损伤SCI进度+5%（价值¥1000） | ¥850 | ⚠️ 机会成本较高，但可立即变现 |

#### 3. 可替代性质疑
**核心问题**：这个任务必须由你亲自做吗？能不能外包/自动化/停止？

**评估方法**：
- 这个任务的核心价值是什么？（决策 vs 执行）
- 如果是执行类任务，能不能外包给AI/助手/外包平台？
- 如果是决策类任务，能不能固化到skill/模板/检查清单？
- 如果完全停止这个任务，会有什么损失？

**输出格式**：
| 任务 | 核心价值 | 可替代方案 | 替代成本 | 是否应该替代 |
|------|---------|-----------|---------|-------------|
| 推文格式调整 | 执行 | 固化到skill | ¥0 | ✅ 已固化 |
| 公众号推文创作 | 决策+执行 | AI生成初稿+人工优化 | ¥0 | ✅ 应该替代 |
| 心肌损伤SCI的GSEA分析 | 决策+执行 | 无法替代（需要专业判断） | - | ❌ 必须亲自做 |

#### 4. 时间投入合理性质疑
**核心问题**：这个任务投入的时间是否合理？是否存在过度优化/完美主义？

**评估方法**：
- 这个任务的"足够好"标准是什么？
- 从80分优化到90分，投入的时间是否值得？
- 是否存在"边际收益递减"（投入越多，收益越少）？

**输出格式**：
| 任务 | 投入时间 | 当前质量 | 足够好标准 | 是否过度优化 | 建议 |
|------|---------|---------|-----------|-------------|------|
| Prompt合集V5.0→V5.1 | 2小时 | 90分 | 80分 | ✅ 过度优化 | 停止迭代V5.2 |
| 推文V1→V2优化 | 3小时 | 85分 | 75分 | ✅ 过度优化 | 下次直接发V1 |
| 心肌损伤SCI的GSEA分析 | 0小时 | 0分 | 80分 | ❌ 投入不足 | 明天至少3小时 |

#### 5. 战略一致性质疑
**核心问题**：这个任务是否符合当前阶段的战略重点？

**评估方法**：
- 当前阶段的战略重点是什么？（5月：主业50%→70% + 申博0%→30%）
- 这个任务是否在战略重点内？
- 如果不在战略重点内，为什么还要做？

**输出格式**：
| 任务 | 当前阶段战略重点 | 是否在重点内 | 如果不在，为什么还做 | 建议 |
|------|----------------|-------------|---------------------|------|
| 心肌损伤SCI | 主业50%→70% | ✅ 在重点内 | - | 继续 |
| 综述1启动 | 申博0%→30% | ✅ 在重点内 | - | 继续 |
| 公众号推文（周中） | 副业（非重点） | ❌ 不在重点内 | 逃避主业困难 | 停止 |
| Prompt合集迭代 | 副业（非重点） | ❌ 不在重点内 | 完美主义 | 停止 |

### 模块三：复利资产识别（新增，每次必做）

   - 复利机制：一次创作，多次使用
2. **系统资产**：skill/模板/工作流/自动化脚本

3. **能力资产**：新学会的技能/方法论
   - 例：python-docx自动化Word操作

4. **关系资产**：新建立的连接/口碑/转介绍
   - 例：帮陈老师学生优化综述（维护申博关系）

**重复决策识别**：

## 飞轮效应引擎（自我增长闭环，每次复盘驱动）

### 核心理念
每天写、每天复盘、每天优化迭代，构成一个完整的自我增长闭环：
**输入 → 输出 → 校准 → 再优化**

当这个循环持续运转，它就变成一个持续自我增强的「人生飞轮」。你输出什么，世界反馈什么；你复盘什么，世界回馈什么；最后，你塑造的不只是内容，而是自己。

### 飞轮三环

**写 → 做：深度转化为产出**
- 每天的工作产出（推文/SCI/综述/skill）是飞轮的动力源
- 产出越多，系统越完善，下次产出越快

**做 → 盘：产出带来反馈**
- 世界的反应（阅读量/客户反馈/导师评价）是最真实的修正信号
- 每日复盘捕捉这些信号，转化为优化方向

**盘 → 写：反思沉淀为洞见**
- 今天的复盘让明天的工作更精准
- 每一次痛苦的反思都在积累认知复利

### 复利三步法（每次复盘强制执行）

**第一步：做一次，用无数次**
- 今天做的工作中，哪些可以做成模板/skill/检查清单？
- 判断标准：如果这件事下次还会做，就必须固化
- 执行：提取方法论 → 写入对应skill → 下次直接调用

**第二步：晒出方法，转化为内容资产**
- 今天解决问题的过程，能不能变成一篇推文/知乎回答？
- 多数人藏着掖着，聪明人懂得分享——分享本身就是复利
- 执行：识别可公开的方法论 → 记入「推文选题池」→ 周末集中创作

**第三步：发现重复，立即升级系统**
- 今天有没有重复做过的事？（上次也做过类似判断/操作）
- 如果有 → 必须写入skill/模板/自动化脚本，不允许第3次手动处理
- 执行：识别重复 → 固化到skill → 更新日志记录

### 飞轮效应检查（每次复盘末尾必须回答）
1. 今天的工作中，有什么可以「做一次用无数次」的？→ 固化到skill
2. 今天解决的问题，能变成一篇内容吗？→ 记入推文选题池
3. 今天有没有重复做过的事？→ 立即升级系统
4. 今天的反馈信号是什么？→ 校准明天的方向

## 长期记忆保存机制（每次复盘后必须执行）

每次复盘完成后，必须将当天复盘的关键内容写入长期记忆系统（`C:\Users\yaoyao\.claude\projects\C--Users-yaoyao\memory\`），确保跨会话可追溯。

### 保存规则
1. **更新或创建 session 记忆文件**：`project_session_{YYYYMMDD}.md`
   - 包含当天所有产出物（文件路径+内容摘要）
   - 各项目当前进度和下一步
   - 关键决策和数据点
   - 复盘中的正向/负向反馈要点
   - 明日行动建议
2. **更新 MEMORY.md 索引**：确保新的 session 记忆有对应条目
3. **按需更新其他记忆**：
   - 项目进度有重大变化 → 更新对应 project 记忆
   - 用户给出新的工作偏好/反馈 → 创建或更新 feedback 记忆
   - 发现新的外部资源引用 → 创建 reference 记忆
4. **不重复保存**：先检查是否已有当天的 session 记忆，有则更新而非新建

### 每日核心洞察提炼→永久记忆（新增，强制执行）

**目的**：每次复盘不只是记录流水账，而是提炼出可跨会话复用的核心洞察，保存为永久记忆。

**执行步骤**：
1. **识别今日核心洞察**：从复盘中提取1-3条「下次还能用」的认知/方法/决策
   - 不是记录「今天做了什么」（那是session记忆的事）
   - 而是提炼「今天学到了什么可以永久复用的东西」
2. **判断是否已有对应记忆**：
   - 如果已有 → 更新/补充
   - 如果没有 → 创建新的feedback/project记忆
3. **写入永久记忆**：
   - 类型选择：方法论/工作偏好 → feedback类；项目决策 → project类
   - 格式：规则 + Why + How to apply
4. **更新MEMORY.md索引**

**判断标准**：
- ✅ 应该保存：「四重奏结构比总分总更适合C类推文」（可复用的方法论）
- ✅ 应该保存：「周中做推文=逃避主业，必须警告」（行为模式识别）
- ❌ 不应该保存：「今天写了一篇推文」（流水账，session记忆已覆盖）
- ❌ 不应该保存：「心肌损伤SCI进度50%」（项目进度，session记忆已覆盖）

**复利逻辑**：每天提炼1-3条洞察 → 30天积累30-90条 → 永久记忆越来越丰富 → AI对用户的理解越来越深 → 每次对话的起点越来越高 → 飞轮加速

### 保存时机
- 复盘 .md 文件生成完毕后立即执行
- 如果用户在复盘过程中补充了新信息，保存时一并纳入
